🐛 Original Error

❌ IndexSizeError

Uncaught IndexSizeError: Failed to execute 'getImageData' on 'CanvasRenderingContext2D': The source width is 0. at tick (scanner.js:224:25)

Cause: The video element wasn't fully loaded when the scanner tried to read from it, resulting in 0 width/height.

🔧 Fixes Applied

📹
Video Ready Check: Added proper video.readyState and dimension validation before processing frames
🛡️
Error Handling: Wrapped tick() function in try-catch to prevent crashes
⏰
Camera Loading: Wait for 'loadeddata' event before starting scan loop
📚
jsQR Library: Added direct jsQR script loading to scanner.html
🔄
Graceful Retries: If video not ready, retry on next animation frame

💻 Code Changes

1. Enhanced Video Ready Check

function tick() { if (!video || !canvas || !ctx || !scanning) return; // Check if video is ready and has valid dimensions if (video.readyState < 2 || video.videoWidth === 0 || video.videoHeight === 0) { // Video not ready yet, try again in next frame requestAnimationFrame(tick); return; } // ... rest of function }

2. Improved Camera Initialization

video.addEventListener('loadeddata', () => { console.log('[CAM] Video loaded, starting scanner...'); setScanStatus('Scanning...', '#fff'); requestAnimationFrame(tick); });

3. Added jsQR Library to HTML

<script src="https://cdn.jsdelivr.net/npm/jsqr@1.4.0/dist/jsQR.js"></script>
